## Описание

Логический сценарий автоматически управляет рекуператором (приточно-вытяжной вентиляцией) в зависимости от показаний датчиков качества воздуха в помещении. Поддерживает одновременную работу двух рекуператоров — в спальне и гостиной — с координацией их режимов в ночное время.

Сценарий привязывается к аксессуару рекуператора, а не к датчику. Каждую минуту он сам опрашивает все подключённые датчики и принимает решение об изменении скорости и режима работы.

---

## Возможности

- **Управление по CO₂** — плавное увеличение скорости притока по мере роста концентрации углекислого газа
- **Управление по VOC и PM** — при повышении летучих органических соединений или взвешенных частиц рекуператор переключается в режим вытяжки
- **Приоритет загрязнителей над CO₂** — если одновременно растут и CO₂, и VOC/PM, приоритет отдаётся вытяжке вредных веществ
- **Ночной режим** — спальня работает на приток, гостиная синхронно на вытяжку с той же скоростью
- **Синхронизация скоростей** — гостиная может работать на той же скорости что и спальня; при выключении спальни переходит на собственные датчики
- **Гистерезис** — скорость снижается только при падении значения ниже нижней границы текущего диапазона, без дёрганий на границе
- **Датчики окон** — при открытом окне рекуператор выключается и не включается автоматически до закрытия
- **Вспомогательные вытяжки** — при критическом уровне VOC/PM включаются вытяжные вентиляторы в ванной и туалете, при нормализации выключаются только если были включены этим сценарием
- **Проверка доступности датчиков** — если датчик не в сети (`C_Online = false`), его показания игнорируются
- **Режим «Не дома»** — при активации охраны в режиме «Не дома» логика полностью отключается
- **Чистые логи** — события пишутся только при реальных изменениях (вкл/выкл/смена скорости/режима)
- **Режим отладки** — опциональное подробное логирование каждую минуту

## Установка

1. Скачайте файл и добавьте MQTT шаблон устройства `Шаблон устройства - MQTT (VAKIO) + Base Smart.json`
2. Добавьте рекуператор в Sprut.Hub
3. Скачайте файл `Рекуператор_v5.4.json`
4. В Sprut Hub перейдите в раздел **Логика → Шаблоны** и импортируйте файл
5. Откройте аксессуар **Рекуператор (спальня)**, добавьте логику из шаблона
6. Заполните параметры (см. раздел ниже), выберите роль **Спальня**
7. Повторите для аксессуара **Рекуператор (гостиная)**, роль **Гостиная**
8. На рекуператоре гостиной включите **Синхронизировать скорость со спальней**

## Параметры настройки

### Основное

| Параметр | Описание |
|---|---|
| **Роль рекуператора** | `Спальня` или `Гостиная`. Определяет какие датчики и логика применяются. Каждый рекуператор настраивается отдельно. |
| **Сигнализация** | Сервис охранной системы Sprut Hub. Режимы: 0 = Дома, 1 = Не дома (логика выключается), 2 = Ночь. |

### Датчики

| Параметр | Описание |
|---|---|
| **Датчик CO₂ спальни** | CarbonDioxideSensor с характеристикой CarbonDioxideLevel. Основной показатель для управления притоком в спальне. |
| **VOC спальни** | AirQualitySensor с характеристикой VOCDensity. При превышении порога — вытяжка. |
| **PM2.5 спальни** | Мелкодисперсные частицы диаметром до 2.5 мкм. При превышении — вытяжка. |
| **PM10 спальни** | Взвешенные частицы диаметром до 10 мкм. При превышении — вытяжка. |
| **Датчик CO₂ гостиной** | Аналог CO₂ спальни для зоны гостиной/кухни. |
| **VOC гостиной** | Летучие органические соединения в гостиной (Qingping Air Monitor 2). |
| **PM2.5 гостиной** | Мелкодисперсные частицы в гостиной. |
| **PM10 гостиной** | Взвешенные частицы в гостиной. |
| **Датчик VOC кухни** | Отдельный датчик VOC на кухне (другая шкала — настройте пороги отдельно). |

### Рекуператоры и вентиляторы

| Параметр | Описание |
|---|---|
| **Этот рекуператор (Fan сервис)** | Fan-сервис аксессуара, к которому привязана логика. Выберите сервис этого же рекуператора. |
| **Рекуператор-партнёр** | Fan-сервис второго рекуператора. Спальня указывает на гостиную и наоборот. Используется для синхронизации скоростей. |
| **Синхронизировать скорость со спальней** | Только для гостиной. При включении гостиная работает на той же скорости что спальня. Ночью: спальня = приток, гостиная = вытяжка. Если спальня выключена — гостиная переходит на собственные датчики. |
| **Вытяжка туалета** | FanBasic с HC.On. Включается при критическом VOC/PM. |
| **Вытяжка ванной — вентилятор 1** | Первый вентилятор ванной комнаты. |
| **Вытяжка ванной — вентилятор 2** | Второй вентилятор ванной (если два в одном аксессуаре — разные сервисы). |

### Датчики окон

До четырёх датчиков открытия окон (ContactSensor). При открытом окне рекуператор выключается и не управляется логикой до закрытия. Логируется только при изменении состояния.

| Параметр | Описание |
|---|---|
| **Датчик окна 1–4** | ContactSensor: открыто = 1, закрыто = 0. Оставьте пустым если датчика нет. |

### Ночной режим

| Параметр | Описание |
|---|---|
| **Начало ночного режима** | Время в формате ЧЧ:ММ, например `23:00`. |
| **Конец ночного режима** | Время окончания, например `07:00`. Поддерживает переход через полночь. |
| **Минимальная скорость ночью — спальня** | Если CO₂/VOC/PM выше нормы, скорость будет не ниже этого значения (1–7). |
| **Минимальная скорость ночью — гостиная** | Аналог для гостиной. |

### Режимы работы VAKIO

Для каждой ситуации можно выбрать один из 7 режимов рекуператора VAKIO:

| Режим | Описание |
|---|---|
| Приток | Подача свежего отфильтрованного воздуха с улицы |
| Приток MAX | Турбо-приток 5 минут, быстрое проветривание |
| Рекуперация лето | Циклический приток/вытяжка с теплообменником, рекомендуется при температуре выше −10°С |
| Рекуперация зима | Рекуперация с удалением наледи, рекомендуется при температуре ниже −10°С |
| Вытяжка | Удаление загрязнённого воздуха из помещения |
| Вытяжка MAX | Турбо-вытяжка 5 минут, быстрое удаление запахов |
| Ночной | 5 минут максимальной вытяжки, затем переход на скорость из настроек приложения |

Настраиваемые ситуации:

| Параметр | Рекомендация |
|---|---|
| **Режим спальни ночью** | Приток |
| **Режим гостиной ночью** | Вытяжка |
| **Режим спальни днём — CO₂** | Приток или Рекуперация |
| **Режим спальни днём — VOC/PM** | Вытяжка |
| **Режим гостиной днём — CO₂** | Приток или Рекуперация |
| **Режим гостиной днём — VOC/PM** | Вытяжка |

### Пороги CO₂ (ppm)

7 порогов скоростей. Рекуператор выключается ниже `co2Off`, включается на скорость 1 выше `co2Spd1` и т.д. Значения по умолчанию основаны на норме CO₂ в жилых помещениях.

| Параметр | По умолчанию | Описание |
|---|---|---|
| CO₂ выкл | 700 ppm | Ниже этого — выключить |
| CO₂ скорость 1 | 800 ppm | Начало вентиляции |
| CO₂ скорость 2 | 950 ppm | Умеренное превышение |
| CO₂ скорость 3 | 1100 ppm | Заметное превышение |
| CO₂ скорость 4 | 1300 ppm | Высокое |
| CO₂ скорость 5 | 1500 ppm | Очень высокое |
| CO₂ скорость 6 | 1700 ppm | Критическое |
| CO₂ скорость 7 | > 1700 ppm | Максимальная |

### Пороги VOC гостиной (мкг/м³)

Основаны на шкале Qingping Air Monitor 2: Отличное 0–65, Хорошее 65–220, Приемлемое 220–660, Низкое 660–2200, Плохое 2200+

### Пороги VOC кухни

Отдельная шкала для датчика кухни (Станция качества воздуха): норма до 37, допустимо до 120, плохо от 293. Настройте под реальные показания вашего датчика.

### Пороги PM2.5 и PM10 (мкг/м³)

Основаны на шкале Qingping: PM2.5 норма до 35, PM10 норма до 54. При превышении — вытяжка.

### Вспомогательные вытяжки

| Параметр | Описание |
|---|---|
| **Включать вытяжки при VOC/PM** | Разрешить управление вентиляторами ванной и туалета. |
| **Минимальная скорость для вытяжек** | Вентиляторы включаются только когда скорость по VOC/PM достигает этого значения (по умолчанию 6 — только при критических показателях). Выключаются только если были включены этим сценарием, не мешая другим автоматизациям. |

### Отладка

| Параметр | Описание |
|---|---|
| **Подробное логирование** | При включении каждую минуту в лог пишется текущее состояние: режим, показания датчиков, принятое решение. Отключите в нормальной работе. |

## Логика приоритетов

```
Не дома → выключить, не управлять
  ↓
Окно открыто → выключить, не управлять
  ↓
Ночное время (по расписанию или режиму охраны)
  ├─ Спальня: VOC/PM > CO₂ → вытяжка
  │           CO₂ > норм  → приток
  │           всё норм    → выкл
  └─ Гостиная (синхр.): скорость = скорость спальни, режим = вытяжка
             (своя)    : CO₂/VOC/PM по своим датчикам
  ↓
Дневное время
  ├─ Спальня: VOC/PM > CO₂ → вытяжка
  │           CO₂ > норм  → приток
  │           всё норм    → выкл
  └─ Гостиная: VOC/PM ≥ CO₂ → вытяжка + вытяжки ванной при скорости ≥ 6
               CO₂ > норм  → приток
               всё норм    → выкл
```

## Динамика скорости

Снижение скорости происходит только при гистерезисе: когда значение опускается ниже **нижней границы текущего диапазона**, а не просто ниже верхней. Это исключает дёргание скорости на границах.

## Совместимость

- Sprut Hub (Nashorn JS, ES5)
- VAKIO Base Smart (MQTT)
- Qingping Air Monitor 2 (CO₂, VOC, PM2.5, PM10)

- Любой датчик VOC/PM с сервисом `HS.AirQualitySensor`
- Датчики окон с сервисом `HS.ContactSensor`
