# 🔧 Фильтр рекуператора VAKIO BASE SMART — логический сценарий для SprutHub

Логический сценарий для умного дома **SprutHub**, который отслеживает ресурс фильтра рекуператора **VAKIO BASE SMART** на основе реального объёма прокачанного воздуха, режима работы, скорости вентилятора, условий установки и сезонности.

Данные **персистентны** — сохраняются в характеристиках устройства (`C_GasMeter`) и переживают перезапуск сценария и хаба.

## Что умеет сценарий

- Считает объём воздуха раздельно по режимам: приток, вытяжка, рекуперация
- Рассчитывает раздельный износ механики F7 и угольного слоя
- Учитывает тип района, этаж и сезон через коэффициенты
- Сохраняет все данные в устройстве — переживает любой перезапуск
- Восстанавливает данные при старте с подробным логом
- Поддерживает несколько рекуператоров — каждый экземпляр сценария независим
- Все сервисы хранения выбираются из выпадающего списка — не зависит от имён

## Принцип расчёта

### Накопление объёма

Сценарий срабатывает при каждом изменении режима или скорости рекуператора. При каждом срабатывании:

1. Закрывается предыдущий интервал — считается `м³ = поток(м³/ч) × время(ч)`
2. Открывается новый интервал
3. Объём распределяется по режимам:
   - **Приток / Вытяжка** — весь объём в соответствующий счётчик
   - **Рекуперация** — 50% в приток, 50% в вытяжку (реверс каждые 40 сек по паспорту)

Страховочный **cron раз в час** фиксирует накопленное если режим не менялся долго.

### Реальный ресурс фильтра

```
Реальный ресурс = Базовый ресурс ÷ (К_район × К_этаж × К_сезон)
```

**Пример** для оживлённой дороги (50–150 м), 6 этаж, лето:
```
F7 реальный = 60 000 ÷ (1.25 × 1.00 × 1.05) ≈ 45 714 м³
Уголь реальный = 30 000 ÷ (1.25 × 1.00 × 1.05) ≈ 22 857 м³
```

### Коэффициент нагрузки на F7 по режиму

| Режим | Коэффициент | Пояснение |
|---|:---:|---|
| Приток | 1.00 | Полная нагрузка |
| Приток MAX | 1.30 | Турбо-режим |
| Рекуперация лето | 1.00 | Объём уже ×0.5 при распределении |
| Рекуперация зима | 1.00 | Аналогично |
| Вытяжка | **0.00** | Наружный воздух через фильтр не проходит |
| Вытяжка MAX | **0.00** | Аналогично |
| Ночной | 0.50 | 5 мин вытяжка + тихий приток |

**Угольный слой** изнашивается только на притоке — на вытяжке уголь не расходуется.

## Архитектура хранения данных

Все данные хранятся в характеристиках устройства и переживают перезапуск хаба:

| Сервис в шаблоне | Тип | Характеристика | Хранит |
|---|---|---|---|
| Воздух всего | `C_GasMeter` | `C_CubicMeter` | м³ всего через фильтр |
| Воздух приток | `C_GasMeter` | `C_CubicMeter` | м³ на притоке |
| Воздух вытяжка | `C_GasMeter` | `C_CubicMeter` | м³ на вытяжке |
| Воздух рекуперация | `C_GasMeter` | `C_CubicMeter` | м³ в режиме рекуперации |
| Износ F7 | `C_GasMeter` | `C_CubicMeter` | % износа механики F7 (0–100) |
| Износ угля | `C_GasMeter` | `C_CubicMeter` | % износа угольного слоя (0–100) |
| Механика F7 | `FilterMaintenance` | `FilterLifeLevel` | Отображение остатка F7 % |
| Угольный слой | `FilterMaintenance` | `FilterLifeLevel` | Отображение остатка угля % |
| Фильтр общий | `FilterMaintenance` | `FilterLifeLevel` + `ResetFilterIndication` | Общий остаток + кнопка сброса |

> `C_GasMeter` — персистентное хранилище, переживает перезапуск.
> `FilterMaintenance` — только отображение в текущей сессии, после перезапуска восстанавливается из `C_GasMeter`.

## Установка

### Шаг 1 — Импортировать шаблон устройства

Импортировать `VAKIO_Base_Smart_with_Filter_v5.json` как шаблон устройства в SprutHub.

### Шаг 2 — Импортировать логический сценарий

Импортировать `Filter_Monitor_v4.1.json` как логический сценарий.

### Шаг 3 — Настроить сценарий

В настройках сценария заполнить в разделе **ОСНОВНОЕ**:

**Рекуператор и параметры фильтра:**
1. **Рекуператор (Fan-сервис)** — выбрать из списка Fan-сервис рекуператора VAKIO
2. **Дата установки фильтра** — формат `ДД-ММ-ГГГГ`, например `26-06-2026`
3. **Базовый ресурс F7** — выбрать из таблицы выше по типу района
4. **Базовый ресурс угля** — аналогично
5. **Тип района** — выбрать из списка
6. **Этаж** — номер этажа

**Сервисы хранения данных** (выбрать из выпадающего списка устройства VAKIO):

| Опция | Выбрать сервис |
|---|---|
| Сервис хранения износа F7 | Износ F7 |
| Сервис хранения износа угля | Износ угля |
| Сервис хранения м³ всего | Воздух всего |
| Сервис хранения м³ притока | Воздух приток |
| Сервис хранения м³ вытяжки | Воздух вытяжка |
| Сервис хранения м³ рекуперации | Воздух рекуперация |
| Сервис отображения ресурса F7 | Механика F7 |
| Сервис отображения ресурса угля | Угольный слой |
| Сервис общего ресурса и сброса | Фильтр общий |

**Производительность по скоростям** — заполнены паспортными значениями, менять не нужно для VAKIO BASE SMART.

## Мониторинг

### Где смотреть данные

- **Карточка устройства** — сервисы «Механика F7», «Угольный слой», «Фильтр общий», счётчики м³
- **Лог автоматизаций** — строки `[Фильтр STATUS]` и `[Фильтр КОЭФФ]` при включённом логировании

### После замены фильтра

1. Физически заменить фильтр
2. В настройках логического сценария включить переключатель **«🔄 Сбросить счётчик фильтра»**
3. Изменить скорость или режим рекуператора чтобы разбудить сценарий
4. Сценарий обнулит все счётчики м³ и износ, запишет 100% в ресурс фильтров, обновит дату установки на сегодня, переключатель сброса выключится автоматически

## Несколько рекуператоров

Создать отдельный экземпляр сценария для каждого устройства. В каждом экземпляре выбрать свои сервисы хранения из соответствующего устройства. В логах каждый сценарий идентифицируется именем комнаты:

```
[Фильтр] [Гостиная / Рекуператор] ...
[Фильтр] [Спальня / Рекуператор] ...
```

## Известные особенности SprutHub

- **`variables` не персистентны** — сбрасываются при перезапуске сценария или хаба. Все данные хранятся в характеристиках устройства (`C_CubicMeter`).
- **`HC.FilterLifeLevel` read-only** в стандарте HomeKit — `setValue` обновляет отображение в текущей сессии, но не сохраняется в БД. Поэтому износ хранится в `C_GasMeter`, а `FilterMaintenance` используется только для отображения.
- **Имена сервисов не должны содержать спецсимволов** — СХ некорректно обрабатывает `³`, скобки и другие символы при `getName()`. В v4.x эта проблема решена — поиск идёт по UUID из `options`, а не по имени.
- **`ResetFilterIndication` не будит сценарий** — платформа не поддерживает подписку на кастомные характеристики через `sourceServices`. Сброс реализован через опцию-переключатель в настройках сценария.
