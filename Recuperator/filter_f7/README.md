# 🔧 Фильтр рекуператора VAKIO BASE SMART — логический сценарий для SprutHub

Логический сценарий для умного дома **SprutHub**, который отслеживает ресурс фильтра рекуператора **VAKIO BASE SMART** на основе реального объёма прокачанного воздуха, режима работы, скорости вентилятора, условий установки и сезонности.

---

## Зачем это нужно

Производитель рекомендует менять фильтр раз в 6 месяцев, но реальный ресурс сильно зависит от:
- режима работы (приток, вытяжка, рекуперация)
- скорости вентилятора
- этажа и расстояния до дороги
- сезона (весенняя пыльца, летняя пыль, зимний чистый воздух)

Сценарий считает **раздельный износ** двух слоёв фильтра:
- **Механика F7** — задерживает пыль, пыльцу, PM2.5
- **Угольный слой** — адсорбирует газы, выхлопы, запахи

Это важно: угольный слой исчерпывается быстрее на притоке у оживлённой дороги, а механика F7 живёт дольше на высоких этажах где крупных частиц меньше.

---

## Принцип расчёта

### Накопление объёма

Сценарий срабатывает **при каждом изменении режима или скорости** рекуператора. При каждом срабатывании:

1. Закрывается предыдущий интервал — считается объём `м³ = поток(м³/ч) × время(ч)`
2. Открывается новый интервал с текущим режимом и скоростью
3. Объём распределяется по режимам:
   - **Приток / Вытяжка** — весь объём в соответствующий счётчик
   - **Рекуперация** — 50% в приток, 50% в вытяжку (реверс каждые 40 сек по паспорту)

Страховочный **cron раз в час** фиксирует накопленное если режим не менялся долго (ночной режим, длительная рекуперация).

### Реальный ресурс фильтра

```
Реальный ресурс = Базовый ресурс ÷ (К_район × К_этаж × К_сезон)
```

Чем хуже условия — тем меньше реальный ресурс. Это физически понятно и легко проверяется вручную.

**Пример** для оживлённой дороги (50–150 м), 6 этаж, лето:
```
F7 реальный = 60 000 ÷ (1.25 × 1.00 × 1.05) = 45 714 м³
Уголь реальный = 30 000 ÷ (1.25 × 1.00 × 1.05) = 22 857 м³
```

### Износ фильтра F7 по режимам

| Режим | Коэффициент | Пояснение |
|---|---|---|
| Приток | 1.00 | Полная нагрузка — уличный воздух через фильтр |
| Приток MAX | 1.30 | Турбо-режим, выше скорость |
| Рекуперация лето | 1.00 | Объём уже ×0.5, двойного снижения нет |
| Рекуперация зима | 1.00 | Аналогично |
| Вытяжка | **0.00** | Наружный воздух через фильтр не проходит |
| Вытяжка MAX | **0.00** | Аналогично |
| Ночной | 0.50 | 5 мин вытяжка MAX + тихий приток |

### Износ угольного слоя

Уголь изнашивается **только на притоке** — адсорбирует газы из уличного воздуха. На вытяжке внутренний воздух не содержит уличных газов → уголь не расходуется.

| Режим | Нагрузка на уголь |
|---|---|
| Приток / Приток MAX | 100% объёма |
| Рекуперация | 50% объёма (только фаза притока) |
| Вытяжка / Ночной | 0% |

## Установка

### Требования

- **SprutHub** с поддержкой логических сценариев
- Рекуператор **VAKIO BASE SMART** интегрированный через MQTT или облако VAKIO
- Шаблон устройства с сервисом `HS.FilterMaintenance` (входит в комплект)

### Шаг 1 — Обновить шаблон устройства

Импортировать `Шаблон устройства - MQTT (VAKIO) + Base Smart Filter.json` как шаблон устройства в SprutHub. Шаблон добавляет сервис `HS.FilterMaintenance` с тремя характеристиками:

| Характеристика | Тип | Описание |
|---|---|---|
| `HC.FilterLifeLevel` | double, 0–100 | Остаток ресурса в % (обновляется сценарием) |
| `HC.FilterChangeIndication` | int, 0/1 | 0 = норма, 1 = нужна замена |
| `HC.ResetFilterIndication` | int, 1 | Запись 1 → сброс счётчика после замены фильтра |

### Шаг 2 — Импортировать логический сценарий

Импортировать `Filter_Monitor_v3.4.json` как логический сценарий в SprutHub.

### Шаг 3 — Настроить сценарий

В настройках сценария заполнить:

1. **Рекуператор (Fan-сервис)** — выбрать из списка Fan-сервис рекуператора VAKIO
2. **Дата установки фильтра** — в формате `ДД-ММ-ГГГГ`, например `01-06-2025`
3. **Базовый ресурс F7** — выбрать из таблицы выше по типу района
4. **Базовый ресурс угля** — аналогично
5. **Тип района** — выбрать из списка
6. **Этаж** — номер этажа установки рекуператора
7. **Производительность по скоростям** — заполнены паспортными значениями, менять не нужно если используется VAKIO BASE SMART

### Шаг 4 — Включить подробное логирование (опционально)

Включить опцию **«Подробное логирование»** на первые несколько дней. В логе автоматизаций SprutHub будут появляться строки:

```
[Фильтр] [Гостиная / Рекуператор] Новый интервал: Приток скорость=3
[Фильтр] [Гостиная / Рекуператор] Интервал: Приток | скор.3 | 45 мин | +60.0 м³ | F7 +0.131% | Уголь +0.262%
[Фильтр STATUS] [Гостиная / Рекуператор] F7=0.13% | Уголь=0.26% | Приток=60.0м³ | Вытяжка=0.0м³ | Всего=60.0м³ | F7ресурс=45714м³ | УгольРесурс=22857м³ | Дней=1 | Сейчас=Приток | скор.3 | 45 мин | ~60.0 м³
[Фильтр КОЭФФ] [Гостиная / Рекуператор] Район=1.25 | Этаж=1.00 | Сезон=1.05 | Делитель=1.313
```

## Мониторинг

### Где смотреть данные

- **`HC.FilterLifeLevel`** в сервисе `Фильтр` устройства — остаток ресурса 0–100%, обновляется при каждом изменении режима
- **Лог автоматизаций SprutHub** — строки с префиксами `[Фильтр STATUS]` и `[Фильтр КОЭФФ]` содержат все текущие данные
- **Уведомления SprutHub** — стандартные уведомления при достижении пороговых значений `HC.FilterChangeIndication`

### Когда менять фильтр

Сценарий устанавливает `HC.FilterChangeIndication = 1` когда **хотя бы один** из слоёв изношен на 80% и более. SprutHub генерирует стандартное уведомление.

> Угольный слой, как правило, исчерпывается раньше механики F7 — особенно вблизи оживлённых дорог.

### После замены фильтра

1. Заменить физический фильтр
2. В SprutHub записать `1` в характеристику `HC.ResetFilterIndication` устройства
3. Сценарий сбросит все счётчики и обновит дату установки на текущую автоматически
4. Отключить и включить логический сценарий (Ресурс сбрасывается при отлючении логического сценария)

## Внутренние переменные сценария

Сценарий хранит состояние между запусками в `variables`:

| Переменная | Описание |
|---|---|
| `m3Intake` | Накопленный объём на притоке (м³) |
| `m3Exhaust` | Накопленный объём на вытяжке (м³) |
| `m3Recup` | Накопленный объём в режиме рекуперации (м³) |
| `m3Total` | Суммарный объём через фильтр (м³) |
| `wearF7` | Расчётный износ механики F7 (%) |
| `wearCarbon` | Расчётный износ угольного слоя (%) |
| `intervalStart` | Timestamp начала текущего интервала |
| `intervalMode` | Режим работы текущего интервала |
| `intervalSpeed` | Скорость текущего интервала |

## Совместимость

Сценарий использует только **ES5-совместимый JavaScript** без современных методов (`padStart`, `includes`, стрелочные функции и т.д.) — движок SprutHub имеет ограниченную поддержку ES6.

Протестировано на:
- VAKIO BASE SMART (MQTT + облако)
- SprutHub (версия на момент разработки: июнь 2026)
