# ❄️ Кондиционер — Температура v3.2

Логический сценарий для **SprutHub**. Управляет кондиционером Lytko по температуре в комнате — включает, выключает и плавно регулирует скорость вентилятора. Учитывает присутствие людей, открытые окна, время суток, уличную температуру и ручное управление.

> Сценарий работает с минимальным набором — **кондиционер + датчик температуры + пороги**. Все остальные датчики и переключатели опциональны: если не задан — условие считается выполненным автоматически.

## Принцип работы

Каждую минуту (и при изменении состояния кондиционера) сценарий проходит по цепочке проверок. Если условие не выполнено — кондиционер выключается и дальнейшие проверки не идут.

| # | Проверка | Что происходит |
|---|----------|----------------|
| 1 | **Зима / Лето** | Если переключатель «Зима» включён — вся логика остановлена |
| 2 | **Режим охраны** | «Не дома» → выключить. «Ночь» → ночные пороги. «Дома» → стандартный режим |
| 3 | **Открытые окна** | Хотя бы одно открыто → немедленное выключение |
| 4 | **Уличная температура** | Холоднее порога → не включать |
| 5 | **Присутствие дома** | Никого из жильцов дома → не включать |
| 6 | **Присутствие в комнате** | Никого в комнате → таймер задержки → выключить |
| 7 | **Температура и скорость** | Включить/выключить по порогам, пересчитать скорость |
| 8 | **Дисплей** | Днём ON, ночью OFF |

## Требования

- SprutHub с поддержкой логических сценариев
- Кондиционер с интеграцией через **Thermostat-сервис** (HC.TargetHeatingCoolingState, HC.C_FanSpeed)
- Внешний **датчик температуры** в комнате (TemperatureSensor)

---

## Установка

1. Скачать файл `Кондиционер_Температура_v3.2.json`
2. В SprutHub → Автоматизация → Логические сценарии → Импорт
3. Привязать сценарий к **Thermostat-сервису** кондиционера
4. Заполнить обязательные настройки (см. ниже)
5. Включить сценарий

> ⚠️ Для каждого кондиционера задаётся **своя** логика сценария.

## Настройки

### 🏠 Основные

| Параметр | Тип | Описание |
|----------|-----|----------|
| **Комната** | Список | Название для логов: Спальня, Гостиная, Детская, Кабинет, Другая |
| **Кондиционер (Thermostat)** | Сервис | Thermostat-сервис кондиционера — основная точка управления |
| **Датчик температуры комнаты** | Сервис | TemperatureSensor — по его показаниям принимаются все решения |

### 🌡️ Температурные пороги

| Параметр | По умолчанию | Описание |
|----------|:------------:|----------|
| **Целевая температура (день)** | 24°C | Передаётся в TargetTemperature кондиционера |
| **Целевая температура (ночь)** | 24°C | То же, для ночного режима |
| **Порог ВКЛЮЧЕНИЯ — день** | 26°C | При достижении — кондиционер включается |
| **Порог ВЫКЛЮЧЕНИЯ — день** | 25°C | При снижении до этого — кондиционер выключается |
| **Порог ВКЛЮЧЕНИЯ — ночь** | 26°C | Аналог для ночного режима |
| **Порог ВЫКЛЮЧЕНИЯ — ночь** | 25°C | Аналог для ночного режима |

> Разница между порогом включения и выключения — это гистерезис. Рекомендуется не менее 1°C чтобы избежать частых циклов включения/выключения.

### 💨 Ступени скорости

Скорость пересчитывается **каждую минуту** по текущей температуре. Задаёте верхнюю границу для каждой ступени — при превышении скорость переходит на следующую.

| Ступень | Название | Порог по умолчанию |
|---------|----------|--------------------|
| 1 | Тихо | < 26°C |
| 2 | Медленно | < 27°C |
| 3 | Средне | < 28°C |
| 4 | Быстро | < 29°C |
| 5 | Турбо | ≥ 29°C |

Отдельный набор порогов для **ночного режима** (те же 5 ступеней).

### 🌙 Ночной режим

| Параметр | По умолчанию | Описание |
|----------|:------------:|----------|
| **Начало ночного режима** | 23:00 | Формат ЧЧ:ММ |
| **Конец ночного режима** | 08:00 | Формат ЧЧ:ММ |

Ночной режим также активируется при режиме охраны «Ночь» (SecuritySystemCurrentState = 2).

### 👥 Присутствие

| Параметр | Тип | Описание |
|----------|-----|----------|
| **Присутствие человека 1** | OccupancySensor | Персональный датчик жильца (напр. Андрей) |
| **Присутствие человека 2** | OccupancySensor | Персональный датчик жильца (напр. Олеся) |
| **Присутствие человека 3** | OccupancySensor | Третий жилец (опционально) |
| **ДП в комнате** | OccupancySensor | Датчик присутствия в данной комнате |

Кондиционер включается только если **хотя бы один** из персональных датчиков показывает «дома» **и** ДП в комнате зафиксировал присутствие.

### ⏱️ Задержка выключения при уходе из комнаты

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|:------------:|----------|
| **Задержка включена** | Boolean | ON | Вкл/выкл задержки |
| **Задержка (мин)** | Integer | 5 | Сколько минут ждать после ухода |

Если задержка **выключена** — кондиционер выключается сразу при исчезновении присутствия.  
Если задержка **включена** — ждёт N минут и выключает. Полезно чтобы не выключаться при кратковременном выходе.

### 🪟 Датчики окон

До **4 датчиков** ContactSensor на комнату. Если хотя бы одно окно открыто — кондиционер немедленно выключается. При закрытии всех окон — возобновляет работу.

| Параметр | Описание |
|----------|----------|
| **Датчик окна 1–4** | ContactSensor. Если не задан — считается закрытым |

### 🌤️ Уличная температура

| Параметр | По умолчанию | Описание |
|----------|:------------:|----------|
| **Датчик уличной температуры** | — | TemperatureSensor (опционально) |
| **Минимальная уличная температура** | 15°C | Ниже этого — кондиционер не включается |

### ❄️ / ☀️ Переключатель Зима/Лето

Switch-сервис. **ON = зима** (логика выключена), **OFF = лето** (логика работает). Удобно отключить кондиционирование на зиму без удаления сценария. Если не задан — считается лето.

### 🔒 Сигнализация

SecuritySystem-сервис хаба СХ. Если не задан — считается режим «Дома».

| Состояние | Значение | Поведение |
|-----------|:--------:|-----------|
| Дома | 0 | Стандартный режим |
| Не дома | 1 | Кондиционер выключен |
| Ночь | 2 | Ночные пороги и скорости |

### 🖥️ Дисплей кондиционера

Switch-сервис дисплея. **Днём** — ON, **ночью** — OFF. Идентификатор сервиса смотреть в «прищепке» аксессуара в SprutHub. Если не задан — не управляется.

### 📈 Динамическое ускорение

| Параметр | По умолчанию | Описание |
|----------|:------------:|----------|
| **Динамика включена** | ON | Boolean ползунок |
| **Минут без снижения** | 15 | Если температура не падает N минут — скорость +1 |

При снижении температуры скорость автоматически возвращается к расчётному значению по текущей температуре.

## Ручное управление

Сценарий распознаёт ручное выключение кондиционера и не включает его обратно автоматически в течение заданной паузы.

| Параметр | По умолчанию | Описание |
|----------|:------------:|----------|
| **Пауза после ручного выключения (мин)** | 30 | Сколько минут не трогать кондиционер после ручного выключения |

**Поведение:**
- Выключили вручную → сценарий ждёт N минут → возобновляет автоуправление
- `0` = ждать пока температура не опустится до порога выключения (tOff)
- Включили вручную в период паузы → пауза снимается, сценарий берёт управление

> Сценарий также защищён от **MQTT-реверта** — когда кондиционер через MQTT сообщает хабу своё физическое состояние и тем самым перезаписывает команду выключения. При обнаружении реверта команда выключения повторяется автоматически.

## Диагностика

| Параметр | Тип | Описание |
|----------|-----|----------|
| **Диагностический лог** | Boolean | Включать только при отладке — спамит каждую минуту |

При включённой диагностике в лог пишется:

```
[DEBUG] [Спальня] Т=25.9°C tOn=26 tOff=25 ACstate=0 wantedOn=false lastMode=0 (ночь)
[DEBUG] [Спальня] Т=25.9°C пороги=[26,27,28,29] база=тихо boost=0 итог=тихо (ночь)
[DEBUG] trigger: source=C[391.13.16 TargetHeatingCoolingState '0' Выключено] value=0
```

Помогает понять: почему кондиционер не включается, какую скорость считает сценарий, кто именно изменил состояние устройства.

## История версий

### v3.х
- Пауза после ручного выключения по таймеру (настраиваемое количество минут)
- Защита от MQTT-реверта с учётом ручного управления
- Распознавание ручного выключения по значению триггера и lastState
- Таймер выключения при уходе из комнаты — теперь запускается только когда кондиционер реально работает
- Исправлен баг повторного включения после выключения (использование lastState вместо физического ACstate)
- Boolean-ползунок для включения/выключения задержки выключения
- Диагностический лог как отдельный ползунок
- Пороги температуры переведены в Integer (совместимость с UI SprutHub)

### v2.x
- 5-ступенчатая плавная регулировка скорости
- Скорость пересчитывается каждый цикл при работающем кондиционере
- Логирование только при изменении событий (окно, присутствие, режим)
- Название комнаты на русском языке в логах
- Поддержка до 4 датчиков окон
- Поддержка уличной температуры
- Переключатель Зима/Лето
- Дисплей через Switch-сервис
